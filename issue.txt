## Overview

Implement the Warehouse Operations module for managing day-to-day warehouse activities including put-away, picking, packing, shipping, and replenishment. Supports multi-task operations with bin-level tracking, pick list optimization by bin location order, and productivity metrics.

## Schemas

- `OperationCreate` - operation_number, warehouse_code, operation_type, reference_type, reference_number, assigned_to, priority?, notes?, created_by, tasks: list[TaskCreate]
- `OperationUpdate` - assigned_to?, priority?, notes?
- `OperationResponse` - all WarehouseOperation fields + id, created_at, updated_at, tasks: list[TaskResponse]
- `TaskCreate` - task_number, product_code, product_name, quantity, from_bin?, to_bin?
- `TaskResponse` - all OperationTask fields + id, created_at, updated_at
- `PickListResponse` - operation_id, operation_number, warehouse_code, tasks: list[TaskResponse sorted by from_bin], total_items, total_quantity
- Productivity response: dict with {operations_per_hour, completion_rate, total_operations, completed_operations, avg_completion_time_minutes}

## Validators

- `operation_type` must be one of: put_away, pick, pack, ship, replenish
- For move operations (put_away, replenish): `from_bin` must not equal `to_bin`
- `quantity` must be > 0 on all tasks
- `operation_number` must be unique
- Status transitions: pending -> in_progress -> completed, pending -> cancelled
- Can only update operations in "pending" status
- Can only cancel operations in "pending" status
- All tasks must be completed before completing the operation
